hexo+github：从零开始搭建个人博客

发表于2023-07-21|更新于2023-07-21|教程

前言：个人博客的重要性

  1. 知识分享和总结: 通过博客，程序员可以分享自己的学习和工作经验，总结技术知识和解决问题的方法。这不仅有助于自己巩固学习成果，还可以帮助其他人理解和学习相关的知识。
  2. 建立个人品牌: 个人博客是一个展示个人技术水平和专业能力的平台。通过不断地发布高质量的技术文章，可以建立自己的个人品牌，提升在技术领域的影响力。
  3. 拓展社交圈: 博客是一个社交平台，可以吸引来自全球各地的读者和其他程序员。通过与读者互动和交流，可以结识志同道合的人，拓展社交圈。
  4. 职业发展: 个人博客是展示自己技术能力的有效方式，可以在找工作或升职时增加竞争力。雇主和招聘者可能会通过阅读你的博客来评估你的技术水平和工作能力。
  5. 学习与成长: 经常写博客要求对技术进行深入思考和研究，这将促使你持续学习和成长。同时，阅读其他技术人员的博客也可以获得新的知识和灵感。
  6. 记录项目和经验: 在博客中记录自己的项目经验和解决问题的过程，有助于回顾和查找。随着时间的推移，博客将成为一个丰富的知识库，帮助你避免重复工作。
  7. 激励与分享: 在博客上分享自己的学习和技术成果，也会激励其他人。你的博客可能帮助其他程序员解决问题或找到新的解决方案，成为其他人的学习资源。

**tips:**博客目录结构如下：

  • _config.yml ：全局配置文件，比如网站名称，副标题，秒速，作者，语言等等。具体可以参考官方文档。
  • scaffolds ：骨架文件，是生成新页面或者新博客的模版。可以根据需求编辑，当hexo生成新博客的时候，会用这里面的模版进行初始化。
  • source ： 这个文件夹下面存放的是网站的markdown源文件，里面有一个_post文件夹，所有的.md博客文件都会存放在这个文件夹下。现在有一个hello-world.md文件。
  • themes ：网站主题目录，hexo有非常丰富的主题支持，主题目录会存放在这个目录下面。更多的主题参见：https://hexo.io/themes/
