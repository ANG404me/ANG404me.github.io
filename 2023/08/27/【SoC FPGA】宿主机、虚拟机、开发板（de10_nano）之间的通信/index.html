【SoC FPGA】1.宿主机、虚拟机、开发板（de10_nano）之间的通信

发表于2023-08-27|更新于2023-08-28|SoC FPGA
|阅读量:

一、实现功能

  • 宿主机、虚拟机、开发板互相通信。

  • 虚拟机和宿主机可以上网。

二、准备工作

  • 需要准备：

    • 开发板要插上烧录了linux映像文件的sd卡，拨码开关位置正确。

    • 两根网线，一个千兆交换机。

    • 一根USB Mini-B线。

  • 操作系统：

    • 宿主机：win11
    • 虚拟机：ubuntu16.04
    • 开发板：Poky 8.0

  • 软件：

    • VMware 16 pro
    • MobaXterm（windows）：用于ssh连接

三、实现步骤

3.1 连线

  • 电脑——网线——交换机——网线——开发板
  • 电脑——USB——开发板的UART-to-USB Controller（靠近以太网口的那个）

3.2 桥接模式/NAT模式/Host-Only模式

vmware为我们提供了三种网络工作模式，它们分别是：Bridged（桥接模式）、NAT（网络地址转换模式）、Host-Only（仅主机模式），虚拟网卡分别对应VMnet0（不显示）、VMnet8、VMnet1。

  • 桥接模式：主机网卡与虚拟机虚拟的网卡利用虚拟网桥进行通信。在桥接的作用下，类似于把物理主机虚拟为一个交换机，所有桥接设置的虚拟机连接到这个交换机的一个接口上，物理主机也同样插在这个交换机当中，所以所有桥接下的网卡与网卡都是交换模式的，相互可以访问而不干扰。**在桥接模式下，虚拟机ip地址需要与主机在同一个网段，如果需要联网，则网关与DNS需要与主机网卡一致。**其网络结构如下图所示：

  • NAT（地址转换模式）：如果你的网络ip资源紧缺，但是你又希望你的虚拟机能够联网，这时候NAT模式是最好的选择。NAT模式借助虚拟NAT设备和虚拟DHCP服务器，使得虚拟机可以联网。在NAT模式中，主机网卡直接与虚拟NAT设备相连，然后虚拟NAT设备与虚拟DHCP服务器一起连接在虚拟交换机VMnet8上，这样就实现了虚拟机联网。VMware Network Adapter VMnet8虚拟网卡主要是为了实现主机与虚拟机之间的通信。

  • Host-Only模式：Host-Only模式其实就是NAT模式去除了虚拟NAT设备，然后使用VMware Network Adapter VMnet1虚拟网卡连接VMnet1虚拟交换机来与虚拟机通信的，Host-Only模式将虚拟机与外网隔开，使得虚拟机成为一个独立的系统，只与主机相互通讯。如果要使得虚拟机能联网，我们可以将主机网卡共享给VMware Network Adapter VMnet1网卡，从而达到虚拟机联网的目的。其网络结构如下图所示：

3.3 配置过程

① 添加虚拟网络VMnet0

在windows搜索里搜虚拟网络编辑器，以管理员身份运行。安装虚拟机后在里面应该有VMnet1和VMnet8这两个网络，这两个网络是不用动的，但要记住VMnet8的子网地址。如果没有VMnet0，则：

添加网络—>选择VMnet0—>选择桥接模式—>外部连接选择以太网网卡—>应用

② 配置以太网网卡静态ip

打开windows控制面板—>网络和internet—>网络和共享中心—>更改适配器设置，可以看到安装虚拟机后多出了两张虚拟网卡：VMware Network Adapter VMnet1和VMware Network Adapter WMnet8。这两个网卡不需要更改，我们主要更改以太网网卡：

右键以太网—>属性—>双击Internet协议版本4（TCP/IPv4）—>把自动获得ip地址改为使用下面ip地址—>修改ip地址为192.168.199.104，子网掩码为255.255.0.0—>确认。

③ 虚拟机添加VMnet0网卡

退出虚拟机，选择编辑虚拟机设置—>添加网络适配器—>选择自定义：特定虚拟网络，选择VMnet0—>确定。

④ 检查

打开虚拟机—>检查网络适配器是否打开（右下角），检查网络是否连接（右上角）—>打开终端—>输入ifconfig命令查看当前网络。一般来说这时候有一个NET网络（我这是ens33）和本地网络（lo），如果NET网络显示ipv4地址为192.168.138.x（这是系统分配的），就可以实现主机与虚拟机的通信，且两者都能上网。不为192.168.138.x可以在网络连接的Edit Connections中修改，参考第⑤步，或输入ifconfig ens33 192.168.138.129命令暂时更改。

⑤ 创建桥接网络

在网络连接中选择Edit Connections—> 类型选择Ethernet—>在Ethernet选项中的Device项选择ens37—>在IPv4 Settings里选择手动ip，Address改为192.168.199.103（注意上图的192.168.137.103是不对的），Netmask改为255.255.0.0，Gateway不填—>save—>打开ens37网络，此时用ifconfig命令就可以查看到该网络了

这里要串口连接开发板（烧了镜像的），用你喜欢的串口方式操作开发板：windows下的putty软件或者linux下的minicom软件。这里以minicom（需要配置串口号，波特率，自行搜教程）为例：

四、互联的优势

4.1 用MobaXterm来代替minicom

由于我们配置了开发板的静态ip并实现了与主机的通信，我们可以使用MobaXterm的ssh来代替minicom与开发板交互，优点：

  1. 不需要那根串口线了；
  2. 文件目录一目了然，MobaXterm赢太多了；
  3. 如果是在windows上安装了开发、调试软件可以使用远程调试功能；
  4. 可以用MobaXterm实现主机与开发板之间的文件传输，比命令行方便。

4.2 用MobaXterm的ssh连接虚拟机（用NET或桥接都可以）

优点：

  1. 文件目录一目了然，习惯命令行操作的话MobaXterm用起来比虚拟机终端要顺手；
  2. 方便主机和虚拟机传输文件。

4.3 虚拟机和开发板通信

优点：

  1. 如果是在虚拟机上安装了开发、调试软件可以使用远程调试功能；

  2. 可以用以下命令实现文件传输：scp hello_world.txt root@192.168.199.102:/home/root/test；

    也可以用主机当中介，从而实现虚拟机和开发板间接文件传输。

总结

这种连接方式适用于开发板无需联网的环境，从Windows端和Linux端都可以对板子进行调试。

五、使用共享网络功能实现开发板联网

注意：这种方法适合在windows上做开发，Linux作为可视化的运行测试端，开发板作为调试与实现最终功能的平台。

5.1 实现功能

  • 主机、虚拟机、开发板都可以上网。
  • 主机与虚拟机、主机与开发板通信正常，但是虚拟机与开发板通信时有问题：ping会出现(DUP!)。

5.2 实现步骤

参考Linux网络配置，这篇文章详细说明了网络地址、路由表、连接测试、域名解析、端口检查。

① 共享网络

打开windows控制面板—>网络和internet—>网络和共享中心—>更改适配器设置—>右键无线网卡选择'属性'—>选择‘共享’，将网络共享给以太网网卡—>确定，开启共享后以太网网卡的ip会自动被改为192.168.137.1

② 修改虚拟网络ens37，修改开发板网络

  • 虚拟机：修改虚拟网络ens37如下图所示，其中Netmask显示24代表255.255.255.0—>保存并重启网络

  • 开发板：vi /etc/network/interfaces

③ 路由表配置

虚拟机&开发板：

  • 输入route或route -n命令来查看当前路由。
  • 添加默认路由：route add default gw 192.168.137.1

④ DNS配置

虚拟机在步骤②中已经配过了，本质上是通过配置来生成resolv.conf文件，所以直接修改resolv.conf文件也是一样的。

简单粗暴：一次性修改（配完即生效）

  • 修改resolv.conf文件，vi /etc/resolv.conf，添加nameserver 192.168.137.1

  • 注意：resolv.conf文件在每次重启后会被重写，但目前开发板我还是使用这个方法，解决方法参考Linux系列：Ubuntu虚拟机设置固定IP上网（配置IP、网关、DNS、防止resolv.conf被重写） - Alexia(minmin) - 博客园 (cnblogs.com)
